# moex_portfolio
# 📈 Оптимизация инвестиционного портфеля по Марковицу

Проект по анализу недельных котировок акций Московской биржи и построению портфеля минимального риска. 
Цель проекта — загрузить исторические данные по акциям, рассчитать недельные доходности, построить ковариационную матрицу и найти идеальные веса портфеля, минимизирующие историческую дисперсию.

---

## 1. Данные

В проекте используются CSV-файлы с историческими данными котировок. Каждый файл соответствует одной акции. Например:
* `YNDX_W1.csv`
* `SBER_W1.csv`
* `GAZA_W1.csv`

Для расчётов используются цены закрытия (Close), так как доходности считаются между ценами закрытия соседних периодов. Исходные файлы должны лежать в директории `data/`.

---

## 2. Методика расчёта

Для каждой акции берётся последовательность цен закрытия. Процентная доходность рассчитывается по формуле:

$$r_t = \frac{P_t}{P_{t-1}} - 1$$

где:
* $P_t$ — цена текущего периода;
* $P_{t-1}$ — цена предыдущего периода.

После этого строится ковариационная матрица доходностей. Портфельный риск считается по формуле:

$$w^T \Sigma w$$

где:
* $w$ — вектор весов акций в портфеле;
* $\Sigma$ — ковариационная матрица доходностей.

Оптимизатор решает задачу минимизации риска (через библиотеку `scipy.optimize`):

$$\min w^T \Sigma w$$

при строгих ограничениях:
* $\sum w_i = 1$ (весь капитал распределяется, сумма долей равна 100%)
* $0 \le w_i \le 1$ (короткие позиции запрещены)

## 4. Основные модули

### `data_loader.py`
Отвечает за работу с данными и файловой системой:
* Класс `DataLoader` — загружает все CSV-файлы из директории `data/` и склеивает их в единый Pandas DataFrame для математики.
* Класс `StreamCSVReader` — реализует паттерн «Итератор» для ленивого чтения файлов.
* Функция `stream_filter()` — очищает потоковые данные, оставляя только нужные значения.

### `optimizer.py`
Содержит класс `Optimizer`, отвечающий за математическую модель Марковица:
* `calculate()` — считает процентное изменение цен (`pct_change`).
* `get_covariance_matrix()` — строит ковариационную матрицу (с аннуализацией).
* `optimize_portfolio()` — использует алгоритм SLSQP для поиска оптимальных весов портфеля.

### `main.py`
Точка сборки проекта. Загружает данные, передает их в Оптимизатор, перехватывает возможные ошибки через `try...except`, строит круговую диаграмму (`plt.pie`) и демонстрирует работу ленивого генератора на примере одного файла.

## 5. Обработка данных в pandas

В проекте выполняются следующие шаги:
1. Чтение исходных файлов через `DataLoader`.
2. Очистка от пустых значений (методом `.dropna()`).
3. Расчёт процентных доходностей через встроенный метод `pct_change()`.
4. Построение матрицы ковариации с помощью `.cov()`.
5. Преобразование результатов в удобный словарь (dict) с тикерами акций и их округленными долями.

---

## 6. Потоковая обработка без pandas (Генераторы)

Для демонстрации работы с большими файлами (Big Data), которые не помещаются в оперативную память, реализовано ленивое построчное чтение без `pandas.read_csv`. 

Класс `StreamCSVReader` работает как генератор (`yield`), который выдает строки строго по запросу (функция `next()`). Это позволяет пропускать гигантские массивы данных через трубу фильтров (`stream_filter`), не загружая файл целиком в RAM.

## 8. Пользовательские исключения

В модуле `optimizer.py` определены кастомные исключения для защиты логики программы:
* `DataError` — выбрасывается, если в класс передана пустая таблица (защита от краша вычислений).
* `OptimizationError` — выбрасывается, если алгоритм оптимизации (`scipy`) не смог сойтись или выдал ошибку математики.

Обе ошибки грамотно обрабатываются в главном файле `main.py`.

---

## 9. Декораторы

В проекте используется собственный декоратор `@timer`. 
Он оборачивает все ключевые методы класса `Optimizer`, замеряет время выполнения математических операций с точностью до миллисекунд и выводит отчет в консоль.
